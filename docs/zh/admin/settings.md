# 设置

**设置**选项卡控制整个服务器的 AI 模型和显示偏好。

## AI 模型

| 设置 | 功能 |
|---|---|
| **基础模型** | Trove 使用的 Gemma 4 变体。列表中只显示已下载的模型。 |
| **上下文窗口 (num_ctx)** | 模型一次可以在内存中保存的文本量，以令牌为单位（大约每个令牌对应 ¾ 个词）。较大的值可处理更长的文档，但会使用更多 RAM。 |

更改模型或上下文窗口后，点击**保存并重建**以应用更改。Trove 会重建其内部模型配置；大约需要 30 秒，并在页面上显示进度。

### 选择模型

| 模型 | 有效参数 | 最低 RAM | 音频 | 最适合 |
|---|---|---|---|---|
| `gemma4:e2b` | 2.3B | ~4 GB | 是 | 配置较低的计算机，响应最快 |
| `gemma4:e4b` | 4.5B | ~6 GB | 是 | 均衡——推荐的默认选项 |
| `gemma4:26b` | 4B 激活 (MoE) | ~10 GB | 否 | 更好的质量，速度与 e4b 相近 |
| `gemma4:31b` | 31B 密集 | ~20 GB | 否 | 最高质量，需要高性能计算机 |

!!! tip "音频 Gems 与模型选择"
    只有 `gemma4:e2b` 和 `gemma4:e4b` 支持音频输入。如果切换到不支持音频的模型，使用音频输入的 Gems 将对用户隐藏，直到切换回来。

## 语言

**语言**选择器更改整个 Trove 界面的显示语言，包括用户端的主页和 Gem 运行界面。目前支持：英语、法语、德语、西班牙语、葡萄牙语、中文、意大利语。

## 数据

**数据**部分允许您备份整个 Trove 配置或恢复以前的备份。

### 导出 Bundle

点击**导出 Bundle** 下载单个 ZIP 文件（`trove-bundle.zip`），包含：

- 所有 Gems 及其设置。
- 所有文档文件夹、文档元数据以及每个文档的转换文本。

在进行重大更改之前使用此功能备份配置，或将配置复制到另一个 Trove 实例。

### 导入 Bundle

点击**导入 Bundle** 打开导入对话框。选择从任意 Trove 实例导出的 `.zip` 文件，然后选择导入模式：

| 模式 | 功能 |
|---|---|
| **添加**（默认） | 将 Bundle 与当前数据合并。现有 Gems 和文档保留不变。如果导入项与现有项 ID 相同，则以新 ID 导入（如 `policy-2`）。 |
| **替换** | 删除所有当前 Gems、文档和文件夹，然后从 Bundle 导入所有内容。 |

!!! warning "替换模式不可撤销"
    替换模式在导入前会永久删除所有现有 Gems 和文档。如果想保留当前状态，请先导出备份。

导入成功后，摘要会显示导入了多少 Gems 和文档，以及是否有因 ID 冲突而重命名的项目。

## 局域网 URL

设置选项卡中显示的局域网 URL 是您网络中的用户应该打开的地址。使用**复制**按钮并分享——例如，张贴在公告栏上或通过电子邮件发送。
