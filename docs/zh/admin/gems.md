# 管理 Gems

**Gem** 是具有固定用途的可重用 AI 任务。用户在主页上将 Gems 视为卡片，填写简短表单即可运行。

## 创建 Gem

1. 在管理面板中打开 **Gems** 选项卡。
2. 点击**新建 Gem**。
3. 填写表单：

| 字段 | 功能 |
|---|---|
| **名称** | 显示在 Gem 卡片上。保持简短且具描述性。 |
| **描述** | 可选。显示在名称下方的单行提示。 |
| **色调** | Gem 图标的颜色。使用不同颜色便于用户一眼区分各个 Gem。 |
| **提示模板** | AI 的指令。使用 `{{ variable_name }}` 占位符作为用户填写的字段。 |
| **功能** | 如果任务需要照片或截图，勾选*接受图片输入*。 |
| **输出模式** | *文本*用于普通输出；*结构化 (JSON)* 用于机器可读输出。 |
| **文档访问** | 运行此 Gem 时 AI 可以读取的文档文件夹或单个文件。 |

4. 点击**创建**。

## 编写好的提示模板

模板是 AI 接收到的指令。可以包含任何文本加上占位符：

```
用 {{ language }} 摘要以下文本，使用不超过 5 个要点：

{{ text }}
```

这将为用户创建两个输入字段：*language* 和 *text*。

**技巧：**

- 具体明确。告诉 AI 您想要的确切格式。
- 如果重要，请说明预期的输出语言。
- 保持指令简短——模型在清晰简洁的 prompt 下效果最佳。
- 在与用户分享之前先自己测试 Gem。

## 文档访问

每个 Gem 可以通过 Gem 表单中的文件夹和文档树访问文档库的一部分：

- **文件夹访问** — 勾选文件夹名称旁边的复选框。AI 可以查看该文件夹中的每个文档，包括后来添加的文档。勾选文件夹会自动勾选其中的所有文档。
- **单个文档访问** — 展开文件夹，只勾选您想要的特定文档。部分（但非全部）文档被勾选的文件夹会显示部分（−）指示符。
- **无访问权限**（默认）— 保持所有框未勾选。AI 不会为此 Gem 使用文档库。

当 Gem 具有文档访问权限时，AI 会自己决定是查阅文档还是从自己的知识中回答。

## 编辑和删除

点击 Gem 旁边的**编辑**更改其设置。点击**删除**将其永久移除。没有撤销操作。

!!! warning "删除 Gem"
    已删除的 Gems 无法恢复。尝试打开已删除 Gem 的 URL 的用户将看到错误提示。
